Ти виконуєш одну автономну housekeeping-ітерацію в цьому репозиторії.

## Межі

- Спочатку прочитай `AGENTS.md` і `docs/architecture-map.md`.
- Працюй лише в уже підготовленій окремій гілці й лише з чистого дерева. На `main` або
  `master` не працюй. Запам'ятай назву гілки та початковий SHA й не змінюй їх упродовж ходу.
- Обери рівно одну малу, завершувану задачу. Не починай roadmap-фічу або SDD-пакет.
- Не додавай залежностей, міграцій, нових API чи широкого рефакторингу. Не вигадуй роботу.
- Не перемикай гілку, не виконуй `git push`, не створюй PR і не змінюй GitHub issues.
- Зміни не більше восьми файлів і створи не більше одного локального коміту.

## Де шукати роботу

Перевір джерела в такому порядку й візьми першу безпечну завершувану задачу:

1. Червоний `npm run verify -- --skip-e2e` — якщо причина в коді, а не в середовищі.
2. Звичайні `TODO` або `FIXME` у product-коді.
3. Відкриті GitHub issues із label `housekeeping` через `gh`, якщо CLI встановлений і
   автентифікований. Issues тільки читати.
4. Один малий дефект або спрощення, знайдене під час читання коду, але лише якщо результат
   можна довести тестом або видаленням конкретної дубляції.

Якщо безпечної конкретної роботи немає, не змінюй файли й не створюй коміт. Заверши відповіддю
`NO_WORK`.

## Як виконувати

1. Сформулюй observable outcome і назви вибране джерело роботи.
2. Для зміни поведінки спочатку отримай GOOD RED тест, потім мінімальний GREEN. Для прогалини
   покриття додай characterization test і не змінюй production-поведінку.
3. Дотримуйся scope задачі та онови документи, які вона прямо називає.
4. Запусти task-specific тести, `npm run lint` і `npm run test:fast`. E2E запускай, якщо задача
   торкається браузера. Наприкінці запусти `npm run verify -- --skip-e2e`.
5. Переглянь повний diff. Якщо scope розрісся, ворота червоні або задача неоднозначна — повернись
   до початкового SHA через `git reset --hard` і `git clean -fd`, повідом причину й не коміть.
6. Якщо все зелене, створи рівно один локальний коміт зі змістовною назвою
   `chore(housekeeping): <що покращено>`. Не пуш.
